# 홈페이지 테마 구조 표준화 모델 (Template Standardization Model)

본 문서는 멀티 테넌트(SaaS) 부동산 홈페이지 플랫폼의 템플릿(테마) 개발을 위한 12대 표준 구조를 정의합니다.
이 12대 표준 구조는 데이터베이스 매핑과 관리자 설정의 편의성을 위해 모든 템플릿이 반드시 공통으로 가지는 뼈대입니다.

## 목적
- 템플릿(Theme)을 변경하더라도 고객이 설정한 데이터가 100% 온전히 유지되고 호환되도록 하기 위함.
- 관리자(Admin) 페이지에서 일관된 UI(단일 폼)를 제공하여 사용 편의성과 개발/유지보수 효율을 극대화하기 위함.

---

## 12개 표준 레이아웃 매핑 체계

모든 템플릿(`template01`, `template02`, ...)의 메인 페이지 구성 요소와 레이아웃은 시각적인 디자인(CSS)만 다를 뿐, 아래의 **12개 공통 섹션(컴포넌트)**으로 철저히 나뉘어 매핑됩니다.

| 번호 | 표준 섹션 명칭 | 설명 및 관리자 설정 요소 | 비고 |
|:---:|:---|:---|:---|
| 1 | **헤더 (Header)** | 상단 네비게이션바, 브랜드 로고, 상단바 테마 색상 지정 | |
| 2 | **푸터 (Footer)** | 하단 사업자 정보 (상호, 대표, 주소, 사업자번호 등), 하단 로고 | |
| 3 | **메인페이지 (Hero)** | 메인 배너 이미지, 핵심 슬로건 문구, 추천매물 노출 갯수 | |
| 4 | **지도검색** | 지도를 통한 전체 매물 검색 UI 연결 | 공통 모듈 |
| 5 | **테마검색 (조건별)** | 테마별 분류 (사무실: 역세권/1층/통건물 등, 원룸: 풀옵션/단기임대 등) | 템플릿에 따라 설정값 분리 |
| 6 | **뉴스기사 (미리보기)** | 홈페이지 내외부의 부동산 뉴스 최신 게시글 썸네일/링크 표시 | |
| 7 | **문의남기기 (Form)** | 메인 화면에 바로 노출되는 빠른 상담/의뢰 접수 폼 | |
| 8 | **오시는길 (Location)** | 부동산 중개소의 위치 지도 (네이버/카카오/구글맵 API 좌표 연동) | |
| 9 | **회사소개 (About)** | 별도 서브페이지: 인사말, 연혁, 조직도 이미지 포함 | 공통 모듈 |
| 10 | **1:1 문의게시판** | 서브페이지: 비공개 고객 상담 및 의뢰 내역 열람 게시판 | 공통 모듈 |
| 11 | **자유게시판** | 서브페이지: 부동산 공지사항, 자유글 등의 스레드 게시판 | 공통 모듈 |
| 12 | **SNS 및 부가옵션** | 홈페이지 곳곳에 붙는 플로팅 버튼, 유튜브/인스타/블로그 바로가기 연결 | |

## 향후 작업 (Next Steps)
- 기존 `template01` (사무실/상가) 및 `template02` (원룸/투룸)의 메인 페이지(`page.tsx`) 구조를 위의 순서에 맞게 통일.
- 부족한 섹션 (예: `template01`의 문의남기기, 두 템플릿 공통으로 뉴스기사 미리보기 및 오시는길 등) 컴포넌트를 각각의 디자인 톤앤매너에 맞게 추가 구현.
- 위 구조를 바탕으로 Supabase의 `homepage_settings` (또는 관련 설정 DB 테이블) 스키마를 JSON 타입 등의 유연한 형태로 정립.
