# 일일 작업 및 회의록 요약 (2026-04-21)

## 📌 1. 금일 개발 및 작업 내역 (UI/UX 템플릿 구현)

오늘은 공실뉴스 유료 부동산을 위한 **'독립형 SaaS 템플릿 홈페이지'**의 핵심 View 화면(프론트엔드)들을 설계하고 이식하는 작업을 집중적으로 진행했습니다.

### 1) 지도 검색 페이지 (`/map`) 고도화
*   **상단 네비게이션 통합:** 기존의 파편화된 지도 메뉴를 제거하고, 사이트 공통 로고와 탭이 포함된 `<Header />`를 완전히 통합시켰습니다.
*   **전문 테마 포커싱:** 아파트/원룸 등 불필요한 1차 카테고리를 삭제하고, 분양받는 부동산의 전문 영역인 **'상가·사무실·공장·토지'** 2차 카테고리를 최상단 필터로 승격시켰습니다. ('상가' 기본값 설정)
*   **관심매물 분리:** 'MY관심공실' 버튼을 상단 필터바 우측으로 분리하여 유저 인터페이스를 한층 깔끔하게 정리했습니다.

### 2) 전체 매물 보기 페이지 (`/properties`) 클론 레이아웃 적용
*   참고 사이트(alpaca.lease)의 UI를 바탕으로 **테일윈드 CSS**를 활용해 반응형 그리드 화면을 완벽히 재현했습니다.
*   `<GridPropertyCard />`를 신규 제작하여 마우스 호버(Hover) 시 테두리가 Gold 색상으로 변하고 사진이 확대되는 고급스러운 애니메이션을 추가했습니다.
*   면적, 보증금, 월임대료, 주변정보 스펙을 좌우 양단으로 분리하여 가독성을 극대화했고, 레이아웃의 Max-width를 1440px로 넓혀 와이드하고 시원한 4열 배치를 구현했습니다.

### 3) 임대/임차 의뢰 페이지 (`/request`) 폼 제작
*   사용자 편의를 위해 '임대 의뢰'와 '임차 의뢰'를 스위치할 수 있는 동적 탭을 구현했습니다.
*   입력 폼은 테이블 방식(좌측 회색 배경 필수/선택 라벨 + 우측 흰색 입력창)으로 모던하게 디자인하여 시각적 불편함이 없도록 구성했습니다.

### 4) 독자적인 자체 뉴스 관 (`/news` & `/news/[id]`) 추가
*   헤더 메뉴에서 '회사소개'를 제거하고 **'뉴스기사'**로 변경했습니다.
*   공실뉴스 중앙망(`gongsilnews.com`)의 기사 리스트 및 상세 페이지 디자인을 그대로 차용하여, 템플릿의 헤더와 푸터 안에 깔끔하게 이식했습니다.
*   좌측 뉴스 피드와 우측 **'전체뉴스 많이 본 뉴스'** (붉은색 테두리 포인트 탑재) 사이드바 투톤 레이아웃을 성공적으로 구현했습니다.

## 🏛️ 2. 아키텍처 및 시스템 기획 회의 결론 (SaaS Multi-tenant)

본 홈페이지는 단일 스탠드얼론 사이트가 아니라, 중앙 공실뉴스 DB(Supabase)를 전적으로 공유하며 개별 부동산에 분양되는 **'템플릿 뷰 엔진(View Engine)'**임이 핵심입니다.

### [회의 1] DB 연동과 디자인 다양성 확보
*   **이슈:** 중앙 공실뉴스의 뉴스/매물 디자인과 분양 템플릿 홈페이지의 상세 디자인이 달라도 괜찮은가?
*   **결정:** 전혀 상관없으며 오히려 **적극 권장**됨. DB는 데이터(텍스트, 사진)만 보관하고, 템플릿 레이아웃은 부동산의 개성에 맞춰 다르게 보여지는 Headless/API 분리 방식이 맞음. 

### [회의 2] 향후 기능 구현 전략 (1:1문의, 회사소개, 자유게시판)
향후 모든 신규 기능과 DB 테이블은 여러 부동산이 하나의 DB를 공유하므로 **`tenant_id (부동산 고유 식별자)`** 컬럼을 필수적으로 부착하기로 협의했습니다.

1.  **회사소개 (정적 프로필):**
    *   게시판 형태가 아닌 단일 뷰. 
    *   `agency_profile` 테이블을 생성하여 각 도메인(`tenant_id`)에 맞는 소개글, 연혁, 회사 위치를 하나만 불러와 노출.
2.  **1:1 문의 (비공개 폼):**
    *   `inquiries` 테이블 생성. 고객이 글을 쓰면 중앙 DB에 특정 `tenant_id`가 박힌 채로 저장.
    *   중개사 관리자 페이지에서 본인에게 들어온 문의만 필터링하여 응대.
3.  **자유게시판 (공용 리스트):**
    *   `tenant_boards` 테이블 생성. 공실뉴스 전체 게시판과 분리.
    *   화면 랜더링 시 `WHERE tenant_id = '우리부동산'` 조건을 걸어놓아, 마치 자기 자신들만의 독자적인 DB를 쓰는 것처럼 보이도록 데이터 격리(Data Isolation) 확보.

> **Next Step:** 협의된 SaaS 테넌트 아키텍처를 바탕으로 `tenant_id`를 기반으로 작동하는 회사소개 및 문의게시판의 프론트엔드 연동을 진행할 예정입니다.

### [회의 3] 다중 템플릿(Multiple Templates) 관리 및 깃허브 보관 구조
*   **이슈:** 오피스 템플릿 외 여러 테마가 추가될 예정. 각 템플릿마다 깃허브 레포지토리를 여러 개 만들어서 관리해야 하는가?
*   **결정:** 템플릿 개수가 늘어나더라도 **1개의 깃허브(현재 Repository)에 모두 모아서 관리하는 '동적 테마(Dynamic Theme)' 구조**를 채택하기로 함.
*   **구현 전략:**
    *   현재 작성된 레이아웃은 `src/components/templates/office` 등 별도 폴더로 모듈화하여 관리.
    *   중앙 관리자 페이지(공실뉴스)에서 각 부동산이 적용할 테마(`theme_name`)를 DB에 저장.
    *   사용자가 접속하면 도메인에 매칭되는 `theme_name`을 DB에서 읽어와, 단일 Next.js 앱 내에서 해당 테마의 UI 컴포넌트로 화면 껍데기를 즉시 교체 렌더링.
    *   **장점:** 코어 로직(매물 검색, DB 연동 등)은 한 곳에서 통제하면서 외형만 쉽고 빠르게 스위칭 가능. 별도의 추가 서버나 배포 작업 불필요.

## 🛠️ 3. 오후 추가 작업 내역 (UI 디테일 보완 및 서브도메인 인프라 구축)

### 1) UI/UX 디테일 튜닝
*   **전체 매물 보기 동기화:** 검색 결과 페이지(`/properties`)에서 사용하던 구형 카드를 메인 페이지와 완전히 동일한 `<PropertyCard />` 컴포넌트로 일괄 교체하여 디자인 통일성을 확보했습니다.
*   **헤더 네비게이션 개선:** '회사소개', '자유게시판', '1:1문의' 등 불필요한 상단 메뉴를 삭제하고, 남은 메뉴들을 로고 바로 옆 좌측으로 밀착 배치해 여백 밸런스를 개선했습니다.
*   **지도 검색 화면 헤더 확장:** `/map` 접속 시 상단 헤더 컨테이너를 가로 제한(Max-width) 없이 전체 화면(`w-full`)으로 자동 확장되게 하여, 로고가 화면 맨 좌측에 위치하도록 최적화했습니다.
*   **뉴스 기사 테마 통일:** 뉴스 기사 뷰 영역과 우측 사이드바의 이질적인 파란색/빨간색 포인트를 모두 브랜드 컬러인 **'골드(Gold)'**로 전면 교체했습니다. 또한 둥근 모의 소셜 사이트 링크를 세련된 라인 아이콘(스크랩, 공유, 글자크기, 인쇄)으로 변경했습니다.

### 2) Vercel 와일드카드 도메인 맵핑 및 멀티테넌시 구조 개편
사전 협의된 SaaS 아키텍처 구조를 실제 코드(Next.js App Router)로 구현하는 구조 개편 작업을 수행했습니다.
*   **Vercel 설정 연결:** 프로젝트의 Domains 환경 설정에서 `*.gongsilnews.com` 와일드카드를 성공적으로 Add하여 무제한 서브도메인 생성 우회로를 확보했습니다.
*   **Next.js Middleware 구현 (`src/middleware.ts`):** 사용자가 어떤 도메인(예: `agent1.gongsilnews.com`)으로 접속하든 최전방에서 호스트네임을 캐치해 서브도메인 이름(`agent1`)만을 파싱하고, 브라우저 주소창 변화 없이 내부적으로 경로(Rewrite)를 연동시키는 미들웨어를 구축했습니다. (로컬 환경 접속 시 자동 우회 테스트 처리 포함)
*   **`[tenant_id]` 동적 라우팅 대규모 이식:** 메인, 뉴스, 매물, 지도, 의뢰 등 기존 루트 라우팅 폴더들 전체를 통째로 `src/app/[tenant_id]/` 경로 안으로 안전하게 이동시켰습니다. 
*   이제 하나의 소스 코드가 수십·수백 개의 **개별 회사 전용 홈페이지처럼 독립적으로 구동될 수 있는 최상위 인프라 구축**이 모두 완료되었습니다.

## 🏗️ 4. SaaS 멀티테넌트 상세 아키텍처 및 정책 기획 회의 (야간)

### 1) 프리미엄(Freemium) 비즈니스 모델 및 접근 권한 확정
"서브도메인은 모든 회원에게 개방하되, 요금제에 따라 혜택에 차등을 둔다"는 핵심 비즈니스 모델을 확정했습니다.
*   **무료부동산 (Free):** `특정서브도메인.gongsilnews.com` 무료 사용 개방. 단, 사이트에 **공실뉴스 광고 배너 / 워터마크가 강제 표출**됨 (자연스러운 바이럴 마케팅 유도 목적). 테마 및 로고 커스텀 권한 제한.
*   **공실뉴스부동산 / 공실등록부동산 (Premium):** 공실뉴스 워터마크 완벽 제거, 다양한 고급 디자인 테마 자유 교체, 커스텀 로고 업로드 허가. 홈페이지 온전한 통제권 부여.
*   구현 로직: `[tenant_id]/layout.tsx` 레이아웃 진입로에서 해당 서브도메인 소유자의 요금제 필드를 조회하여, 화면 컴포넌트의 렌더링 범위를 동적으로 제한.

### 2) 관리 제어(어드민)와 뷰 렌더러(템플릿)의 완벽한 역할 분리
*   **관리자 페이지 (Port 3000 / 공실뉴스 본진):** 홈페이지 개설, 테마 선택, 로고 업로드 지정 등 "설정을 변경하는 모든 행위(컨트롤 타워)"는 무조건 기존 공실뉴스 통합 관리자 페이지 내부의 [홈페이지] 메뉴에서만 이루어져야 합니다.
*   **템플릿 서버 (Port 3001 / 현재 프로젝트):** 오로지 DB에 저장된 값에 따라 화면만 그려주는 **쇼룸/뷰 전용 엔진** 역할만 수행하여 코드가 무거워지는 것을 방지합니다. 프론트 홈페이지 내에는 별도의 관리용 로그인 화면이나 설정 창을 절대로 만들지 않습니다.

### 3) 멀티테넌시 확립을 위한 신규 테이블 구조 정의
기존 본진 시스템의 `members`, `vacancies` 테이블 구조를 전혀 훼손하지 않기 위해 아래 2개의 전용 테이블을 신설하여 1:1로 매칭시킵니다.
1.  **`homepage_settings` (홈페이지 껍데기 전용 통제 테이블):**
    *   기존 회원 테이블의 고유 번호를 외래키(`owner_id`)로 연결합니다.
    *   `subdomain`(영문 주소), `theme_name`(테마 값), `logo_url`(헤더 이미지 주소), `company_intro`(회사 소개글) 등을 관리합니다.
2.  **`tenant_boards` (다중 입주사 통합 게시판 테이블):**
    *   게시판 종류가 늘어날 때마다 테이블을 만드는 복잡도를 100% 제거하기 위한 수단입니다.
    *   어느 부동산의 홈페이지에 적힌 글인지(`owner_id`)와 어떤 성격의 게시판인지(`board_type`: NOTICE, FREE, INQUIRY)를 단일 테이블에 모조리 모아, 프론트엔드가 파라미터만 던지면 즉시 자동 필터링 되도록 설계합니다.
