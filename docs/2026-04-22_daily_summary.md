# 📝 2026-04-22 작업 요약 (Vercel 배포 및 트러블슈팅)

오늘(`2026-04-22`)은 로컬에서 완성된 멀티테넌트(Multi-tenant) 아키텍처를 실제 라이브 서버인 **Vercel 운영 환경에 완벽히 연동시키고 발생하는 문제들을 해결**하는 과정을 거쳤습니다.

## ✅ 완료된 주요 작업 및 트러블슈팅 내용

### 1. `middleware.ts` Vercel 미리보기 주소 오류 수정
- **문제점:** Vercel에서 각 배포마다 자동으로 생성해주는 임시 주소(`...vercel.app`)로 접속 시, 실제 브라우저에서는 404 에러가 발생하는 문제.
- **원인:** 운영 도메인(`*.gongsilnews.com`)을 기반으로 파싱하는 로직에서 긴 임시 주소가 루트 도메인 우회(Bypass)로 빠져버림.
- **해결 방안:** Vercel Preview 전용 로직 추가. `hostname.includes('vercel.app')` 식별 시 자동으로 `test-agent`라는 테스트 테넌트로 연결하도록 미들웨어 수정 적용 완료. [코드 커밋/푸시 대기 중]

### 2. PC 환경에서 `ERR_CONNECTION_REFUSED` 발생 문제 해결
- **문제점:** 모바일 환경에서는 정상 접속되는 라이브 URL(`buildon.gongsilnews.com`)이 데스크톱 브라우저에서는 연결 거부 에러를 냄.
- **해결 방안:** 로컬 환경 테스트 목적으로 작성했던 윈도우 `hosts` 파일(`C:\Windows\System32\drivers\etc\hosts`) 내의 강제 라우팅(127.0.0.1) 내역을 추적하여 삭제. 로컬 PC에서도 Vercel 서버로 정상적인 외부 DNS 조회를 하도록 복구 완료.

### 3. Vercel 배포 본의 `/map` 페이지 500 서버 에러 해결
- **문제점:** 메인 페이지는 무사히 렌더링되나, 동적으로 데이터를 가져오는 '지도검색(/map)' 페이지 접근 시 Vercel 500 Server Error가 발생함.
- **원인:** 페이지 내 `getVacancies` Server Action이 실행되면서 Supabase에 접근하려 했으나, Vercel 배포 환경에 DB 접속 키(`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`)가 세팅되어 있지 않아 프로그램이 충돌(Crash).
- **해결 방안:** Vercel 프로젝트 환경변수 설정 창에 로컬의 `.env.local` 키값 6개를 모두 등록하고 **[Redeploy]** 실행하여 정상 작동 확인.

### 4. 카카오 맵(API) 미출력 현상 대응
- **문제점:** 지도 페이지 접근은 성공했으나 지도가 떠야 할 자리가 텅 빈(회색) 상태로 출력됨.
- **해결 방안:** 신규 도입한 운영 도메인(`buildon.gongsilnews.com`)이 카카오 디벨로퍼스(Kakao Developers) 플랫폼 설정의 '허용된 Web 사이트 도메인'에 누락되어 있어서 발생한 차단 조치. 카카오 설정에 `*.gongsilnews.com` 등 와일드카드 도메인을 허용 항목으로 추가하도록 안내 및 조치.

### 5. Office Sample 1 아키텍처 구조 정리
- **현 상황:** 현재 라이브 반영된 `[tenant_id]` 뷰 엔진 서빙 아키텍처는 기획된 **'사무실 테마 (Office Sample 1)'** 기본 구조로 구동 중임을 확인.
- **향후 확장:** DB(`homepage_settings`)의 `theme_name` 값 추가 확장에 따라 테마 폴더/컴포넌트만 이 뼈대 위에 계속 얹으면 화면이 알아서 분기되도록 설계 완성.
